# 2026-03-19 (Day 19) 

## 오늘 한 것

오늘은 VMware 환경에서 Memory 성능 문제를 중심으로 공부했다.

기존에는 CPU와 Disk 위주로 분석했다면  
오늘은 메모리 부족 상황과 ESXi 메모리 압박(Ballooning, Swap)을 함께 보는 연습을 했다.

특히 vmstat의 si/so 값이 단순 I/O가 아니라  
메모리 부족으로 인해 Swap이 발생하는 지표라는 것을 새롭게 이해했다.

또한 ESXi에서의 MEMCTL(Ballooning)과 SWCUR(Swap) 개념을 학습했다.

문제 풀이를 통해  
OS에서는 정상처럼 보이지만 실제로는 Ballooning이 발생하고 있는 상황과,  
Memory 문제로 인해 Disk I/O가 증가하면서 Disk 문제처럼 보이는 케이스를 분석했다.

마지막에는 Memory, Disk, VMware 지표가 섞인 문제를 통해  
실제 원인이 Memory라는 것을 판단하는 연습을 했다.

---

## 기억할 것

- si / so는 Swap 발생 지표 (메모리 부족)
- MEMCTL은 Ballooning (ESXi가 메모리 회수)
- SWCUR는 ESXi Swap
- Memory 문제는 Disk 문제처럼 보일 수 있음
- Memory → Swap → Disk I/O 증가 → 성능 저하

---

## 느낀 점

Memory는 CPU나 Disk보다 더 헷갈리는 영역이라는 느낌이 들었다.

특히 OS에서는 정상처럼 보이는데 실제로는 문제가 발생하는 경우가 있어서  
VMware까지 같이 봐야 정확하게 판단할 수 있다는 것을 느꼈다.

이제는 단순히 “느리다”가 아니라  
그 원인이 CPU인지, Disk인지, Memory인지 그리고 ESXi까지 포함해서 생각해야 한다는 것을 이해하게 된 것 같다.
