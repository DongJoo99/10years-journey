# 2026-03-15 (Day 15) 

## 오늘 한 것

오늘은 서버가 느릴 때 CPU, Memory, Disk 중 어떤 부분이 문제인지 판단하는 연습을 했다.

여러 가지 상황을 가정해서 실제 서버에서 볼 수 있는 지표들을 가지고 원인을 분석하는 방식으로 공부했다.

CPU 사용률, Load Average, vmstat, iostat 값을 같이 보면서 문제의 방향을 판단하는 연습을 했다.

특히 CPU idle이 높은데도 Load Average가 높을 수 있다는 점과  
이런 경우 Disk I/O 대기 때문에 Load가 올라갈 수 있다는 것을 이해하게 되었다.

또 Disk 문제라고 해서 항상 util 값이 높은 것만은 아니라는 것도 알게 되었다.

await 값이 높으면 요청 하나하나의 처리 시간이 길어지는 것이기 때문에  
util이 낮아도 디스크 응답 지연 문제가 발생할 수 있다는 점이 인상적이었다.

여러 상황을 비교해보면서 CPU, Memory, Disk 문제를 구분하는 기준이 조금씩 정리되는 느낌이다.


## 기억할 것

- CPU idle이 낮으면 CPU 병목 가능성
- CPU idle이 높은데 Load가 높으면 Disk I/O 대기 의심
- vmstat wa 값이 높으면 Disk 대기
- Memory available이 낮고 Swap이 사용되면 메모리 부족
- Disk util이 높으면 디스크 포화 상태
- await 값이 높으면 디스크 응답 지연


## 느낀 점

지금까지 배운 내용이 따로따로 있는 것이 아니라 서로 연결되어 있다는 느낌이 들었다.

CPU, Memory, Disk를 각각 보는 것이 아니라 전체 지표를 같이 보고 판단해야 한다는 것을 조금 이해하게 된 것 같다.

아직 완전히 익숙하지는 않지만 이런 방식으로 계속 연습하면 실제 서버 문제도 어느 정도 판단할 수 있을 것 같다는 생각이 들었다.

- 실제 운영에서 나오는 장애 상황 분석
- 지금까지 배운 명령어 다시 복습
