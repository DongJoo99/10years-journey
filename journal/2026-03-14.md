# 2026-03-14 (Day 13)

## 오늘 한 것

오늘은 vmstat과 iostat 명령어를 이용해서 서버 상태를 확인하는 방법을 공부했다.

vmstat 명령어를 통해 CPU, Memory, Disk 상태를 한 번에 확인할 수 있다는 것을 알게 되었고
특히 r, si, so, wa 값을 중심으로 서버 문제의 방향을 판단하는 연습을 했다.

r 값이 높으면 CPU 대기 작업이 많다는 의미이고
si, so 값이 증가하면 Swap이 발생하고 있다는 의미이며
wa 값이 높으면 디스크 I/O 대기가 발생하고 있다는 것을 이해했다.

또 iostat 명령어를 이용해서 디스크 사용률을 확인하는 방법도 배웠다.

r/s, w/s 값을 통해 초당 읽기/쓰기 요청을 확인할 수 있고
%util 값을 통해 디스크 사용률을 판단할 수 있다는 것을 알게 되었다.

실습으로 dd 명령어를 사용해서 실제로 디스크에 파일을 쓰는 작업을 발생시켜 보고
그 상황에서 iostat 값이 어떻게 변하는지도 확인했다.


## 기억할 것

- vmstat은 CPU, Memory, Disk 상태를 한 번에 볼 수 있는 명령어
- r 값은 CPU를 기다리는 작업 수
- si / so 값이 증가하면 Swap 발생
- wa 값이 높으면 디스크 I/O 대기 발생
- iostat %util 값이 높으면 디스크 사용률이 높은 상태


## 느낀 점

CPU, Memory, Disk 문제를 각각 따로 배우는 것이 아니라
서로 연결해서 판단하는 연습을 하니까 조금씩 구조가 보이기 시작했다.

특히 Load Average만 보고 CPU 문제라고 생각하는 것이 아니라
CPU idle, wa, disk util 같은 값들을 같이 보고 판단해야 한다는 점이 인상적이었다.

실제 서버 문제를 분석할 때 이런 식으로 접근해야겠다는 생각이 들었다.
