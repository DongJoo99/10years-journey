# 2026-02-25

## 오늘 한 것
Ubuntu VM에서 리눅스 명령어를 복습했다.
파일과 디렉토리 구조를 다시 확인하면서 상대경로와 절대경로 개념을 정리했다.

단순 명령어보다는 실제 장애 상황을 가정하고 어떻게 확인할지 생각해봤다.
VM이 느려질 때 어떤 순서로 점검해야 하는지도 정리했다.

VMware 환경에서 VM 성능 문제 원인도 생각해봤다.
CPU Ready 개념과 Datastore Latency가 VM 성능에 영향을 줄 수 있다는 것을 이해했다.

## 기억할 것

- CPU Ready는 VM이 CPU를 사용하려고 했지만 Host에서 CPU를 받지 못하고 기다린 시간이다.
- vCPU를 많이 할당하면 오히려 성능이 떨어질 수 있다.
- Datastore Latency가 높으면 VM이 느려질 수 있다.
- VM 문제인지 Host 문제인지 먼저 구분해야 한다.

## 느낀 점

리눅스 명령어 자체는 어렵지 않았지만,
장애 상황을 가정하고 생각하는 과정이 도움이 됐다.

특히 VM이 느릴 때 어디부터 확인해야 하는지
정리할 수 있어서 좋았다.
