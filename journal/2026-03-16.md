# 2026-03-16 (Day 15)

## 오늘 한 것

오늘은 서버가 느려지는 상황을 가정하고 여러 지표들을 보고 원인을 판단하는 연습을 했다.

top, uptime, vmstat, iostat 값을 같이 보면서 CPU, Memory, Disk 중 어디에서 문제가 발생했는지 분석하는 방식으로 진행했다.

특히 Disk 문제라고 해서 항상 util 값이 높은 것만은 아니라는 점이 인상적이었다.

util 값은 낮지만 await 값이 높을 경우 요청 하나의 처리 시간이 오래 걸리기 때문에 디스크 응답 지연이 발생할 수 있다는 것도 알게 되었다.

또 vmstat의 r 값이 높을 경우 CPU run queue가 길어지면서 CPU 대기가 발생할 수 있다는 것도 다시 정리할 수 있었다.


## 기억할 것

- CPU idle이 낮고 r 값이 높으면 CPU 병목 가능성
- CPU idle이 높은데 Load Average가 높으면 Disk 대기 가능성
- vmstat wa 값이 높으면 Disk I/O 대기
- Disk util이 낮아도 await 값이 높으면 응답 지연 가능
- vmstat r 값은 CPU run queue를 의미


## 느낀 점

지금까지 배운 내용이 하나씩 연결되기 시작한 느낌이다.

CPU, Memory, Disk를 각각 따로 보는 것이 아니라 여러 지표를 같이 보고 판단해야 한다는 것을 조금씩 이해하게 되는 것 같다.

아직 완전히 익숙하지는 않지만 이런 방식으로 계속 연습하면 실제 서버 문제도 어느 정도 분석할 수 있을 것 같다는 생각이 들었다.


## 내일 할 것

- CPU / Memory / Disk 혼합 문제 추가 연습
- vmstat / iostat 다시 복습
- 실제 운영 장애 분석 연습
