# 2026-03-10 (Day 10)

## 오늘 한 것

오늘은 실제 운영 상황을 가정한 문제 풀이 방식으로 공부했다.

서버가 느릴 때 CPU, Load Average, Disk 상태를 어떻게 확인해야 하는지
순서를 생각해보는 연습을 했다.

특히 CPU 사용률이 낮아도 Load Average가 높을 수 있다는 점과
이런 경우 디스크 I/O 병목을 의심해야 한다는 것을 다시 확인했다.

또 디스크가 99% 찼을 때 어떤 문제들이 발생하는지와
로그 파일이 디스크를 차지하는 경우를 어떻게 찾는지도 배웠다.

그리고 로그 파일을 삭제했는데도 디스크 용량이 줄지 않는 경우가 있는데
그 이유가 프로세스가 삭제된 파일을 아직 잡고 있는 상태일 수 있다는 것도 이해했다.


## 기억할 것

- CPU 사용률과 Load Average는 다른 개념
- CPU 낮음 + Load Average 높음 → I/O wait 의심
- 디스크 사용량 확인은 df → du 순서로 좁혀가기
- 삭제된 파일을 프로세스가 잡고 있으면 디스크 공간이 반환되지 않을 수 있음


## 느낀 점

지금까지 배운 CPU, Load Average, Memory, Disk 개념들이
조금씩 연결되는 느낌이 들었다.

단순히 명령어를 아는 것보다
서버가 느릴 때 어떤 순서로 확인해야 하는지 생각하는 것이 중요하다고 느꼈다.


## 내일 할 것

- CPU / Load Average 문제 풀이 훈련
- 실제 장애 상황 판단 연습
