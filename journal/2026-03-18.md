# 2026-03-18 (Day 18) 

## 오늘 한 것

오늘은 VMware 성능 분석에서 자주 보는 esxtop 핵심 지표를 공부했다.

기존에는 Linux에서 보이는 CPU, Load Average, vmstat, iostat 위주로 원인을 판단했다면  
오늘은 ESXi 관점의 지표까지 함께 보면서 VM 외부에서 발생하는 병목을 해석하는 연습을 했다.

특히 %RDY, %USED, %CSTP 개념을 새롭게 학습했다.

- %RDY: VM이 CPU를 받고 싶지만 받지 못하고 대기하는 시간
- %USED: 실제 CPU 사용량
- %CSTP: 멀티 vCPU 동기화 대기 시간

또한 Disk 쪽에서는 DAVG, KAVG, GAVG 의미를 다시 정리했다.

- DAVG: 스토리지 자체 지연
- KAVG: ESXi 내부 지연
- GAVG: Guest가 체감하는 전체 지연

문제 풀이를 통해  
CPU idle이 높아도 %RDY가 높으면 CPU Ready 문제일 수 있다는 점,  
await가 높고 util이 낮으면 물리 디스크보다 KAVG를 먼저 의심해야 한다는 점을 연습했다.

마지막에는 CPU와 Disk, VMware 지표가 섞인 종합 문제를 풀면서  
여러 문제가 동시에 보일 때 우선순위를 정하는 연습도 했다.

---

## 기억할 것

- CPU idle 높아도 %RDY가 높으면 CPU Ready 문제 가능
- %CSTP는 멀티 vCPU 동기화 대기 시간
- %RDY는 CPU 자체를 못 받는 대기
- await 높고 util 낮으면 KAVG 의심
- GAVG = DAVG + KAVG
- 여러 지표가 동시에 안 좋을 때는 더 강한 병목부터 우선순위를 잡아야 함

---

## 느낀 점

오늘 공부하면서 Linux 안에서만 보던 시야가 ESXi까지 넓어지는 느낌이 들었다.

특히 CPU 사용률만 보고 CPU가 문제 없다고 생각하면 안 된다는 점이 인상적이었다.  
실제로는 VM이 CPU를 받지 못해서 느릴 수도 있고,  
디스크도 단순히 util만 보면 안 되고 KAVG 같은 내부 지연까지 봐야 한다는 것을 알게 되었다.

이제는 “OS 문제인지, Hypervisor 문제인지”를 같이 생각하는 습관이 조금씩 생기고 있는 것 같다.
