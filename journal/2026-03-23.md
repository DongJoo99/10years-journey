# 2026-03-23 (Day 22)

## 오늘 한 것

오늘은 CPU, Disk, Memory, VMware 지표가 섞인 상황에서
주원인을 정확하게 하나로 판단하는 연습을 했다.

특히 Disk처럼 보이지만 실제로는 Memory 문제인 경우,
CPU처럼 보이지만 실제로는 CPU Ready 문제인 경우,
그리고 vCPU 동기화 문제(%CSTP)까지 포함해서 분석하는 연습을 진행했다.

문제 풀이를 통해  
DAVG가 높으면 물리 스토리지 문제,  
KAVG가 높으면 ESXi 내부 지연,  
SWCUR와 si/so가 보이면 메모리 부족,  
%CSTP가 높으면 vCPU 스케줄링 문제라는 흐름을 확실하게 정리했다.

마지막 문제에서는 CPU, Disk, Memory가 모두 애매한 상황에서  
%CSTP를 주원인으로 판단하는 연습을 했다.

---

## 기억할 것

- %RDY는 CPU를 못 받는 대기
- %CSTP는 vCPU 동기화 대기
- DAVG 높음 → 물리 스토리지 문제
- KAVG 높음 → ESXi 내부 지연
- SWCUR 높음 → ESXi Swap (매우 위험)
- Memory 문제는 Disk 문제처럼 보일 수 있음
- 여러 지표가 동시에 안 좋을 때는 반드시 주원인 1개를 먼저 잡아야 함

---

## 느낀 점

오늘은 단순히 지표를 해석하는 것이 아니라
헷갈리는 상황에서도 하나의 원인을 확신 있게 선택하는 연습을 한 날이었다.

특히 %CSTP처럼 OS에서는 잘 보이지 않는 문제도
실제로는 성능에 큰 영향을 줄 수 있다는 점이 인상적이었다.

이제는 단순히 수치를 보는 것이 아니라
“이게 왜 발생했는지”를 먼저 생각하는 습관이 조금씩 잡히는 것 같다.
