# 2026-03-13 (Day 13) 

## 오늘 한 것

오늘은 VM에서 메모리 부족 상황을 직접 만들어보는 실습을 했다.

python 명령어를 이용해서 일부러 메모리를 계속 사용하는 프로세스를 실행했고
free -h와 top 명령어로 메모리 사용량 변화를 확인했다.

메모리가 부족해지자 Swap 영역이 사용되는 것을 확인할 수 있었다.

Swap이 사용되기 시작하면 RAM에 있어야 할 데이터가 디스크로 이동하게 되고
필요할 때마다 디스크에서 다시 읽어와야 하기 때문에
서버가 느려질 수 있다는 것을 이해했다.

특히 CPU 사용률이 거의 없는 상태에서도
서버가 느려질 수 있다는 상황을 확인할 수 있었다.


## 기억할 것

- RAM이 부족해지면 Swap이 사용된다
- Swap은 디스크이기 때문에 RAM보다 매우 느리다
- CPU 사용률이 낮아도 서버가 느릴 수 있다
- Swap이 계속 증가하면 메모리 문제를 의심해야 한다
- 문제 발생 시 먼저 어떤 프로세스가 메모리를 많이 사용하는지 확인해야 한다


## 느낀 점

CPU와 Load Average 실습을 했을 때도 그렇지만
직접 부하를 만들어보니까 개념이 훨씬 잘 이해되는 느낌이다.

특히 CPU 문제가 아닌데도 서버가 느려질 수 있다는 상황을
직접 확인해볼 수 있어서 좋았다.

CPU, Load Average, Memory, Swap이 서로 어떻게 연결되는지
조금씩 이해가 되는 것 같다.
