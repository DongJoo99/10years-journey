# 2026-03-25 (Day 24)

## 오늘 한 것

오늘은 문제를 맞추는 것보다 면접에서 어떻게 설명하는지에 집중해서 연습했다.

같은 문제라도 단순히 원인을 맞추는 것이 아니라  
문제 유형, 주원인, 판단 근거, 한 줄 요약까지 구조적으로 말하는 연습을 진행했다.

특히 CPU Ready, Disk latency(KAVG), Memory Swap 문제를  
각각 면접에서 설명할 수 있도록 정리했다.

또 util과 await 관계, DAVG와 KAVG 차이,  
Memory 부족 시 Swap이 발생하면서 Disk 문제처럼 보이는 흐름을 다시 정리했다.

마지막 문제에서는 Memory 부족으로 인해 Swap이 발생하고  
그로 인해 서버가 멈추는 상황을 설명하는 연습을 했다.

---

## 기억할 것

- CPU idle 높고 Load 높으면 CPU Ready 의심
- await 높고 util 낮으면 KAVG 의심
- DAVG 높으면 물리 스토리지 문제
- available 메모리가 낮고 si/so 발생하면 Swap
- SWCUR 높으면 매우 위험한 상태
- Memory 부족은 Disk 문제처럼 보일 수 있음

---

## 느낀 점

오늘은 단순히 문제를 푸는 것보다  
어떻게 설명해야 하는지가 더 중요하다는 것을 느꼈다.

같은 내용을 알고 있어도 구조적으로 말하지 못하면  
면접에서는 전달이 잘 되지 않을 수 있다는 생각이 들었다.

이제는 문제를 맞추는 것뿐만 아니라  
“왜 그렇게 판단했는지”를 명확하게 말하는 연습이 필요하다고 느꼈다.
