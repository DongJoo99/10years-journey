# 2026-03-11 (Day 11)

## 오늘 한 것

오늘은 CPU 사용률과 Load Average 개념을 이해하는 데 집중했다.

CPU 사용률이 어떻게 계산되는지와
us, sy, id 값이 각각 어떤 의미를 가지는지 확인했다.

또 Load Average가 단순한 CPU 사용률이 아니라
CPU를 필요로 하는 작업의 개수라는 것을 이해하려고 했다.

특히 CPU 코어 수와 Load Average의 관계를 중심으로
어느 정도부터 성능 저하가 발생하는지 기준을 정리했다.


## 기억할 것

- CPU idle이 낮으면 CPU 사용률이 높은 상태
- Load Average는 CPU를 기다리는 작업 수
- CPU 코어 수가 중요 기준이 됨
- Load < Core → 여유
- Load > Core → 대기 발생


## 느낀 점

처음에는 CPU 사용률과 Load Average가 같은 의미라고 생각했는데
완전히 다른 개념이라는 것을 알게 되었다.

아직 숫자를 보고 바로 판단하는 건 익숙하지 않지만
기준을 알고 나니까 조금씩 보이기 시작하는 느낌이다.


## 내일 할 것

- CPU 부하 실습
- yes 명령어로 CPU 점유 확인
- Load Average 변화 직접 확인
