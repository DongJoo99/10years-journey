# 2026-03-11 (Day 10)

## 오늘 한 것

오늘은 실제 서버 장애 상황을 가정한 문제 풀이 방식으로 공부했다.

CPU 사용률과 Load Average를 같이 보고 서버 상태를 판단하는 연습을 했다.

CPU 사용률이 높고 Load Average가 CPU 코어 수와 비슷한 경우에는
CPU가 거의 포화 상태라는 것을 확인했다.

반대로 CPU 사용률이 낮은데 Load Average가 높은 경우에는
디스크 I/O 대기 상황을 의심해야 한다는 것도 이해했다.

또 디스크 사용률이 99%일 때 로그 파일이 디스크를 많이 차지할 수 있다는 것과
로그 파일을 삭제했는데도 디스크 용량이 줄지 않는 경우
프로세스가 삭제된 파일을 잡고 있을 수 있다는 것도 알게 되었다.

마지막으로 CPU 사용률이 300%처럼 보이는 이유가
CPU 코어 기준으로 계산되기 때문이라는 것도 이해했다.


## 기억할 것

- CPU 사용률과 Load Average는 다른 지표
- CPU 낮음 + Load 높음 → Disk I/O 의심
- CPU 높음 + Load ≈ Core → CPU 포화 상태
- 로그 파일 삭제 후 디스크가 줄지 않으면 프로세스가 파일을 잡고 있을 수 있음
- %CPU는 CPU 코어 기준으로 계산됨


## 느낀 점

CPU, Load Average, Memory, Disk가 서로 연결된다는 느낌이 들기 시작했다.

단순히 명령어를 아는 것보다
서버가 느릴 때 어떤 순서로 확인해야 하는지 생각하는 것이 중요하다고 느꼈다.

아직 완전히 익숙하지는 않지만 반복해서 연습하면
실제 서버 문제를 판단하는 능력이 생길 것 같다는 생각이 들었다.
