# 2026-03-04 (Day 6)

## 오늘 한 것

오늘은 Linux 로그를 확인하는 방법을 공부했다.

journalctl 명령어를 사용해 최근 로그를 확인하는 방법과
특정 서비스 로그를 확인하는 방법을 실습해봤다.

특히 ssh 서비스 로그를 확인하면서
서비스가 종료되고 다시 시작된 기록을 시간 기준으로 확인했다.

또 일부러 존재하지 않는 서비스(sshx)를 재시작해보면서
systemctl status 명령어로 에러 메시지를 확인하는 연습도 했다.

로그를 길게 보는 것보다
원인을 알려주는 한 줄을 찾는 것이 중요하다는 것을 느꼈다.


## 기억할 것

- journalctl -n → 최근 로그 확인
- journalctl -u → 특정 서비스 로그 확인
- journalctl -f → 실시간 로그 확인
- systemctl status → 서비스 상태 + 로그 확인
- 로그에서는 시간 / 프로세스 / 메시지를 먼저 본다


## 느낀 점

로그를 보는 방법을 처음으로 제대로 연습해봤다.

생각보다 로그가 길어서 처음에는 조금 막막했지만
한 줄씩 의미를 나눠서 보니까 이해가 되기 시작했다.

아직 다양한 로그를 본 것은 아니라서
더 많은 케이스를 보면 좋을 것 같다는 생각이 들었다.
