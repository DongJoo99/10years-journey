# 2026-03-17 (Day 16)

## 오늘 한 것

오늘은 기존 Linux 성능 분석에서 한 단계 더 나아가 VMware 환경까지 포함해서 분석하는 연습을 했다.

지금까지는 CPU, Memory, Disk 지표만 보고 판단했다면
오늘은 “OS는 정상인데 왜 느릴까”라는 관점으로 문제를 접근했다.

특히 CPU idle이 높은데도 Load Average와 r 값이 높은 상황을 통해
CPU 자체 문제가 아니라 VM이 CPU를 받지 못하는 상황(CPU Ready)을 의심해야 한다는 것을 배웠다.

또한 Disk 분석에서 await와 util을 같이 보는 것이 중요하다는 것을 다시 정리했다.

* util 높음 + await 높음 → 실제 디스크 병목
* util 낮음 + await 높음 → ESXi 내부 지연 가능성 (KAVG)

그리고 VMware Disk Latency 구조인 DAVG, KAVG, GAVG 개념을 처음 학습했다.

---

## 기억할 것

* CPU idle 높음 + r 높음 → CPU Ready 의심
* Load Average는 실제 CPU 사용률이 아니라 “대기 포함” 값
* await는 디스크 응답 시간 (20ms 이하 정상 기준)
* util 낮고 await 높으면 디스크 자체가 아니라 ESXi 문제 가능성
* GAVG = DAVG + KAVG
* Linux await는 VMware의 GAVG에 해당

---

## 느낀 점

Linux 지표만 볼 때는 어느 정도 판단이 가능했는데
VMware가 들어오니까 같은 지표도 완전히 다르게 해석된다는 걸 느꼈다.

특히 CPU가 놀고 있는데도 작업이 밀리는 상황이 이해가 안됐는데
CPU Ready 개념을 알고 나서 왜 그런지 이해가 됐다.

이제부터는 단순히 “CPU 문제, Disk 문제”가 아니라
“이게 VM 문제인지 Hypervisor 문제인지”까지 같이 생각해야 한다는 걸 느꼈다.
