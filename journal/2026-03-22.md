# 2026-03-22 (Day 21)

## 오늘 한 것

오늘은 여러 지표가 동시에 안 좋게 보이는 상황에서 가장 큰 원인을 하나로 정리하는 연습을 했다.

특히 Disk, CPU, Memory가 동시에 문제처럼 보일 때
겉으로 보이는 현상만 보는 것이 아니라
어떤 지표가 주원인인지 우선순위를 정하는 데 집중했다.

문제 풀이를 통해  
KAVG가 높으면 ESXi 내부 I/O 지연,  
SWCUR가 높으면 메모리 부족으로 인한 심각한 Swap 상태,  
DAVG가 높으면 물리 스토리지 지연이라는 흐름을 다시 정리했다.

또한 물리 디스크 지연이 심한 경우
Disk 문제만 나타나는 것이 아니라
Load Average와 run queue도 함께 올라가면서 CPU도 같이 문제처럼 보일 수 있다는 점을 이해했다.

오늘은 단순히 “어디가 이상한지”가 아니라
“가장 먼저 봐야 할 원인이 무엇인지”를 고르는 연습을 한 날이었다.

---

## 기억할 것

- DAVG 높음 → 물리 스토리지 지연
- KAVG 높음 → ESXi 내부 I/O 지연
- SWCUR 높음 → ESXi Swap 발생
- Memory 부족은 서버가 멈추는 것처럼 느껴질 수 있음
- Disk 지연이 심하면 CPU도 같이 안 좋아 보일 수 있음
- 여러 지표가 동시에 안 좋을 때는 주원인 1개를 먼저 잡아야 함

---

## 느낀 점

오늘은 여러 문제가 동시에 보일 때
한 가지 원인으로 정리하는 것이 생각보다 어렵다는 걸 느꼈다.

특히 CPU 값도 안 좋고 Disk 값도 안 좋으면
둘 다 문제라고 말하기 쉬운데,
실제로는 더 큰 원인을 먼저 찾는 것이 중요하다는 걸 알게 되었다.

이제는 단순히 수치를 보는 것보다
“이 값이 왜 같이 나왔는가”를 생각하는 습관이 조금씩 생기는 것 같다.
