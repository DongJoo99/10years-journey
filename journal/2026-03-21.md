# 2026-03-21 (Day 20)

## 오늘 한 것

오늘은 Linux와 VMware 지표를 함께 보고 실제 원인을 하나로 정리하는 연습을 했다.

특히 Disk 문제처럼 보이지만 실제로는 스토리지 자체 문제인지,
ESXi 내부 문제인지,
아니면 Memory 부족으로 인해 Disk처럼 보이는 상황인지를 구분하는 데 집중했다.

문제 풀이를 통해 DAVG가 높으면 물리 스토리지 지연,
%RDY가 높으면 CPU Ready,
si/so와 MEMCTL, SWCUR가 보이면 Memory 부족과 Swap을 먼저 의심해야 한다는 흐름을 다시 정리했다.

또한 사용자가 “CPU를 별로 안 쓴다”라고 느끼는 상황도
실제로는 VM이 CPU를 못 받고 대기하고 있는 CPU Ready 문제일 수 있다는 점을 이해했다.

오늘은 여러 지표가 동시에 보일 때
겉으로 보이는 현상보다 실제 원인을 하나로 좁히는 연습을 했다.

---

## 기억할 것

- DAVG 높음 → 물리 스토리지 지연
- KAVG 높음 → ESXi 내부 지연
- %RDY 높음 → VM이 CPU를 받지 못하고 대기
- si / so 발생 → Guest Swap
- MEMCTL → Ballooning
- SWCUR → ESXi Swap
- Memory 부족은 Disk 문제처럼 보일 수 있음

---

## 느낀 점

오늘은 단순히 지표를 해석하는 것보다
여러 문제가 동시에 보일 때 가장 큰 원인을 하나로 정리하는 것이 더 중요하다는 걸 느꼈다.

특히 Disk가 느려 보여도
실제로는 Memory 부족 때문에 Swap이 발생해서 그렇게 보일 수 있다는 점이 인상적이었다.

조금씩 “어떤 값이 보이면 무엇을 먼저 의심해야 하는지” 기준이 잡혀가는 느낌이다.
