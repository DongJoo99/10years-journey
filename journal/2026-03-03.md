# 2026-03-03 (Day 5)

## 오늘 한 것

여행 이후 다시 공부를 재개했다.

오늘은 Host Not Responding 알람이 발생했을 때
어떻게 판단해야 하는지 구조적으로 정리해봤다.

Host Not Responding이 반드시 호스트가 죽은 상황은 아니라는 점을 이해했다.
vCenter와 ESXi의 관리 통신 문제일 가능성이 더 높다는 것도 알게 되었다.

HA가 언제 동작하고 언제 동작하지 않는지도 생각해봤다.
Datastore Heartbeat가 살아있다면 Host는 실제로 살아있는 것으로 판단되어
VM이 재기동되지 않을 수 있다는 점이 인상적이었다.

Isolation 개념과 Split Brain 상황도 처음으로 제대로 이해했다.
같은 VM이 두 곳에서 동시에 실행될 수 있다는 점이 위험하다는 것을 알게 되었다.

Migration과 Restart의 차이도 다시 정리했다.
HA는 마이그레이션이 아니라 재기동이라는 점을 명확히 이해했다.


## 기억할 것

- Host Not Responding ≠ Host Down
- HA는 vCenter가 아니라 클러스터 기준으로 판단한다
- Datastore Heartbeat는 Host 생존 판단에 중요하다
- HA는 Migration이 아니라 Restart
- Split Brain을 방지하기 위해 HA는 보수적으로 동작한다


## 느낀 점

아직 완전히 정리된 느낌은 아니지만
생각하면서 구조가 연결되는 과정이 재미있었다.

단순히 기능을 아는 것보다
왜 그렇게 동작하는지 이해하는 게 더 중요하다는 생각이 들었다.

오늘은 복귀 첫날이라 무리하지 않고
감각을 다시 잡는 느낌으로 마무리했다.
