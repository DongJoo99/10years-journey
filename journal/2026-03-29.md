# 2026-03-29 (Day 27)

## 오늘 한 것

오늘은 실제 면접 상황을 가정하고 빠르게 문제를 판단하고 답변하는 연습을 했다.

기존에는 충분한 시간을 가지고 분석했다면  
오늘은 제한된 시간 안에서 문제 유형, 주원인, 즉시 조치, 한 줄 요약까지  
빠르게 정리해서 말하는 훈련을 진행했다.

특히 OS 지표는 정상인데 서비스가 느린 경우  
애플리케이션(DB connection pool) 문제일 수 있다는 점을 다시 확인했다.

또한 메모리 부족으로 Swap이 발생하고 OOM이 발생했을 때  
단순 분석이 아니라 서비스 재기동을 통해 먼저 복구해야 한다는 점을 정리했다.

마지막으로 특정 시간대에만 발생하는 성능 문제는  
CPU Ready나 vCPU 동기화 문제(%CSTP)와 함께  
스케줄 작업이나 부하 집중을 같이 의심해야 한다는 것을 이해했다.

---

## 기억할 것

- connection pool exhausted는 DB 문제가 아니라 애플리케이션 문제일 수 있음
- OOM 발생 시 분석보다 서비스 복구가 우선
- Memory 부족 → Swap → 서비스 지연 및 종료
- %CSTP는 vCPU 동기화 문제 (vCPU 많을수록 발생 가능)
- 특정 시간대 문제는 스케줄/배치 작업 의심
- 문제 유형 → 주원인 → 즉시 조치 → 한 줄 요약 구조 유지

---

## 느낀 점

오늘은 문제를 맞추는 것보다  
얼마나 빠르게 판단하고 정확하게 말할 수 있는지가 중요하다는 것을 느꼈다.

같은 내용을 알고 있어도  
짧고 명확하게 전달하지 못하면 의미가 없다는 생각이 들었다.

특히 실무에서는 분석도 중요하지만  
서비스를 먼저 살리는 것이 더 중요하다는 점이 인상적이었다.

이제는 단순히 공부하는 단계가 아니라  
실제 상황에서도 대응할 수 있는 방향으로 성장하고 있다는 느낌이 든다.
