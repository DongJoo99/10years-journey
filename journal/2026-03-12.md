# 2026-03-12 (Day 12) 

## 오늘 한 것

오늘은 VM에서 직접 CPU 부하를 만들어보는 실습을 했다.

yes 명령어를 이용해서 CPU를 계속 사용하는 프로세스를 만들고
top과 uptime 명령어를 통해 CPU 사용률과 Load Average가 어떻게 변하는지 확인했다.

CPU 코어 수를 먼저 확인하고 yes 프로세스를 하나씩 늘려가면서
CPU 사용률과 Load Average의 변화를 관찰했다.

특히 CPU 코어보다 더 많은 작업이 실행될 경우
프로세스가 번갈아가면서 CPU를 사용하게 되고
Load Average가 CPU 코어 수보다 커질 수 있다는 것을 직접 확인했다.


## 기억할 것

- CPU 1코어는 100%로 계산된다
- CPU 사용률은 코어 기준으로 합산된다
- Load Average는 CPU를 필요로 하는 작업의 평균 개수
- Load < CPU Core → CPU 여유
- Load ≈ CPU Core → CPU 거의 포화
- Load > CPU Core → CPU 대기 발생


## 느낀 점

CPU와 Load Average의 관계를 이론으로만 이해하는 것보다
직접 부하를 만들어보면서 확인하니까 훨씬 이해가 잘 되었다.

특히 yes 프로세스를 여러 개 실행했을 때
CPU 사용률이 어떻게 나뉘는지와
Load Average가 왜 증가하는지 직접 볼 수 있어서 좋았다.

지금까지 배운 CPU, Load Average 개념이 조금씩 연결되는 느낌이 들었다.
