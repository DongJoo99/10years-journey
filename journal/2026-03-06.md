# 2026-03-06 (Day 8)

## 오늘 한 것

오늘은 CPU 사용률과 Load Average의 차이를 공부했다.

처음에는 Load Average가 CPU 사용률이라고 생각했지만
실제로는 CPU를 사용하려고 기다리는 프로세스 수라는 것을 알게 되었다.

특히 CPU 코어 수와 Load Average를 비교해서
서버 상태를 판단하는 방법을 이해했다.

또 CPU 사용률이 낮아도 Load Average가 높을 수 있다는 것도 알게 되었고
이런 경우에는 디스크 I/O 대기를 의심해볼 수 있다는 것을 배웠다.

CPU idle이 높아도 서버가 느릴 수 있다는 점이 인상적이었다.


## 기억할 것

- Load Average는 CPU 사용률이 아니다
- Load Average는 CPU를 사용하려고 기다리는 작업 수
- Load Average < CPU Core → 여유
- Load Average = CPU Core → CPU 꽉 참
- Load Average > CPU Core → 대기 발생

그리고 CPU 사용률이 낮아도 서버가 느릴 수 있다.


## 느낀 점

처음에는 Load Average 개념이 조금 어려웠지만
CPU 코어 수와 비교해서 생각하니까 이해가 되기 시작했다.

특히 CPU가 놀고 있는데도 서버가 느려질 수 있다는 부분이 흥미로웠다.

앞으로 서버 상태를 확인할 때 CPU 사용률만 보는 것이 아니라
Load Average도 같이 확인해야겠다고 생각했다.
