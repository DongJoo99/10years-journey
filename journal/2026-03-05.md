# 2026-03-05 (Day 7)

## 오늘 한 것

오늘은 Linux 프로세스에 대해 공부했다.

ps와 top 명령어를 사용해서 현재 서버에서 어떤 프로세스들이 실행되고 있는지 확인하는 방법을 실습했다.

특히 PID와 PPID의 의미를 이해하면서 프로세스가 부모와 자식 구조로 만들어진다는 것도 알게 되었다.

yes 명령어를 사용해서 CPU를 계속 사용하는 프로세스를 직접 만들어보고 top에서 어떻게 보이는지도 확인했다.

그리고 kill 명령어로 해당 프로세스를 종료하는 실습도 해봤다.

또 백그라운드 실행(&)이 어떤 의미인지도 직접 실행해보면서 이해했다.

이후에는 실제 운영 상황을 가정해서 CPU 사용률이 높은 프로세스를 발견했을 때 바로 kill 하면 안 되는 이유와 먼저 확인해야 하는 것들에 대해 생각해봤다.

마지막으로 디스크가 99% 이상 찼을 때 어떤 문제가 발생할 수 있는지와 디스크 사용량을 추적하는 방법도 같이 이야기했다.


## 기억할 것

- ps : 프로세스 목록 확인
- top : CPU / 메모리 사용률 실시간 확인
- PID : 프로세스 ID
- PPID : 부모 프로세스 ID
- kill : 프로세스 종료
- & : 백그라운드 실행

그리고 서비스 프로세스는 바로 kill 하지 말고 먼저 어떤 서비스인지 확인해야 한다.


## 느낀 점

프로세스 개념은 알고 있다고 생각했는데 실제로 PID와 PPID를 같이 보면서 이해하니까 구조가 더 명확하게 보였다.

특히 운영 상황을 가정해서 문제를 어떻게 확인해야 하는지 생각해보는 과정이 재미있었다.

단순히 명령어를 외우는 것보다 왜 확인해야 하는지 생각하면서 배우는 게 훨씬 이해가 잘 되는 것 같다.

- Load Average가 무엇을 의미하는지 이해
